API IFA | DOCUMENTACION 


El proceso de ventas tiene 3 etapas en orden consecutivo, 

1 Orden de Venta (pedido)

2 Entrega (salida de mercancias)

3 Facturacion 

Cada una de estas operaciones debe ir referenciada a la anterior.

/docs

Método: GET

Resumen: Obtener documentación de la API

/orden

Método: POST

Descripción: Permite crear una nueva orden en el sistema.

Request headers:

{
   "authorization": "Token otorgado por IFA"
  }

Request Body:

Campos:

  • DocDate: string - Fecha del documento en formato "YYYY-MM-DD" (ejemplo: "2024-08-22")
  • DocDueDate: string - Fecha de vencimiento en formato "YYYY-MM-DD" (ejemplo: "2024-08-22")
  • CardCode: string - Código del cliente (ejemplo: "C000004")
  • Reference1: string - Referencia 1 (opcional)
  • Reference2: string - Referencia 2 (opcional)
  • Comments: string - Comentarios del documento (opcional)
  • JournalMemo: string - Memo del diario (ejemplo: "Deliveries - C000004")
  • PaymentGroupCode: number - Código de grupo de pago (ejemplo: "-1" para contado)
  • SalesPersonCode: number - Código del vendedor (ejemplo: 35)
  • TransportationCode: number - Código de transporte (ejemplo: -1)
  • ContactPersonCode: number - Código de la persona de contacto (ejemplo: 91479)
  • Series: number - Serie del documento (ejemplo: 221)
  • TaxDate: string - Fecha de impuestos en formato "YYYY-MM-DD" (ejemplo: "2024-08-22")
  • ShipToCode: string - Código de envío (ejemplo: "Z/ CHANCADORA-ENTRE RIOS")
  • FederalTaxID: string - ID de impuesto federal (ejemplo: "3535107")
  • DiscountPercent: number - Porcentaje de descuento (ejemplo: 0.0)
  • PickStatus: string - Estado de selección (ejemplo: "tNO")
  • Pick: string - Selección (ejemplo: "tNO")
  • TrackingNumber: string - Número de rastreo (opcional)
  • PickRemark: string - Observaciones de selección (opcional)
  • U_NIT: string - NIT del cliente (ejemplo: "0")
  • U_RAZSOC: string - Razón social (ejemplo: "n/a")
  • U_ICE: number - Importe ICE (ejemplo: 0.0)
  • U_EXENTO: number - Importe exento (ejemplo: 0.0)
  • U_OSLP_ID: number - ID del representante de ventas (opcional)
  • U_Sucursal_Interna: string - Sucursal interna (opcional)
  • U_DESCUENTO: number - Descuento adicional (ejemplo: 0.0)
  • DocumentLines: array - Lista de líneas del documento
    • LineNum: number - Número de la línea (ejemplo: 0)
    • ItemCode: string - Código del artículo (ejemplo: "101-001-001")
    • Quantity: decimal - Cantidad del artículo (ejemplo: 1.0)
    • Price: decimal - Precio antes de impuestos (ejemplo: 0.0)
    • PriceAfterVAT: decimal - Precio después de impuestos (ejemplo: 0.0)
    • DiscountPercent: decimal - Porcentaje de descuento aplicado (ejemplo: 0.0)
    • VendorNum: string - Número del proveedor (opcional)
    • SerialNum: string - Número de serie (opcional)
    • WarehouseCode: string - Código del almacén (ejemplo: "100-12")
    • SalesPersonCode: number - Código del vendedor (ejemplo: 35)
    • CommisionPercent: decimal - Porcentaje de comisión (ejemplo: 0.0)
    • BaseType: number - Tipo base del documento relacionado (ejemplo: -1)
    • BaseEntry: number - Entrada base del documento relacionado (ejemplo: 0)
    • BaseLine: number - Línea base del documento relacionado (ejemplo: 0)
    • MeasureUnit: string - Unidad de medida (ejemplo: "CAJA X 1")
    • UnitsOfMeasurment: decimal - Unidades de medida (ejemplo: 1.0)
    • UnitPrice: decimal - Precio unitario (ejemplo: 0.0)
    • UoMEntry: number - Entrada de unidad de medida (ejemplo: 2)
    • UoMCode: string - Código de unidad de medida (ejemplo: "CAJA X 1")
    • InventoryQuantity: decimal - Cantidad de inventario (ejemplo: 1.0)
    • RemainingOpenInventoryQuantity: decimal - Cantidad de inventario restante abierta (ejemplo: 1.0)
    • ParentLineNum: number - Número de la línea padre (opcional)
    • TransportMode: number - Modo de transporte (ejemplo: 0)
    • U_COSTO_COM: decimal - Costo del artículo (ejemplo: 0.0)

/entrega

Método: POST

Descripción: Permite crear una nueva entrega de venta en el sistema.

Notas: Cada entrega debe referenciar con el valor de "orderNumber" de las ordenes creadas. 

Request Body:

Campos:

  • DocDate: string - Fecha del documento en formato "YYYY-MM-DD" (ejemplo: "2024-08-22")

  • TaxDate: string - Fecha de impuestos en formato "YYYY-MM-DD" (ejemplo: "2024-08-22")
  • ShipToCode: string - Código de envío (ejemplo: "Z/ CHANCADORA-ENTRE RIOS")
  • FederalTaxID: string - ID de impuesto federal (ejemplo: "3535107")

    • BaseLine: number - Línea base del documento relacionado igual que LineNum (ejemplo: 0)

    • UnitsOfMeasurment: decimal - Unidades de medida (ejemplo: 1.0)
